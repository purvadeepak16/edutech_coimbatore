## Summary

This diagram shows how all components work together to create a seamless text-to-speech experience:

1. **Frontend** collects user input
2. **Backend** orchestrates API calls and data storage
3. **TTS Service** generates high-quality audio
4. **Firestore** persists all data
5. **Browser** plays the audio for the user

All communication is secured with Firebase authentication and proper error handling ensures a robust user experience.
